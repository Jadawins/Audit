Détails techniques

Cette page explique en détail comment fonctionne Audit MS, quelles données sont accédées, quels rôles sont nécessaires et quelles permissions Microsoft Graph sont demandées lors de la connexion.

Comment ça fonctionne

Audit MS utilise l'API Microsoft Graph pour interroger votre tenant Microsoft 365 en lecture seule. L'authentification est gérée par Microsoft directement via OAuth 2.0 — vos identifiants ne transitent jamais par nos serveurs.

L'analyse est entièrement effectuée dans votre navigateur. Les résultats sont stockés en sessionStorage et effacés automatiquement à la fermeture de l'onglet.

Rôles requis sur le tenant
Recommandé
Administrateur général (Global Administrator)
Accès complet à tous les modules. Peut approuver les permissions Graph pour l'ensemble du tenant lors de la première connexion.
Alternatif
Lecteur général + Administrateur Intune (Global Reader + Intune Administrator)
Permet d'accéder aux modules Entra ID, Intune et Microsoft 365. Certaines données peuvent être partiellement restreintes selon la configuration du tenant.
Les modules pour lesquels le compte n'a pas les droits suffisants afficheront N/A au lieu d'un score. L'outil reste utilisable partiellement.
Lors de la première connexion, Microsoft demandera un consentement administrateur pour les permissions ci-dessous. Ce consentement peut être accordé une fois pour tous les utilisateurs du tenant.
Permissions Microsoft Graph demandées

Ces permissions sont demandées lors de la connexion OAuth. Elles sont toutes en lecture seule — aucune ne permet de modifier des données sur votre tenant.

User.Read.AllLire la liste de tous les utilisateurs et leurs attributs (nom, email, date de dernière connexion)
Directory.Read.AllLire l'annuaire Azure AD complet (groupes, unités organisationnelles)
Policy.Read.AllLire les politiques d'accès conditionnel (Conditional Access)
AuditLog.Read.AllLire les journaux de connexion et d'audit (utilisé pour détecter les comptes inactifs)
Organization.Read.AllLire les informations du tenant (nom, domaines vérifiés, identifiant)
RoleManagement.Read.AllLire les rôles d'administration et leurs membres (ex : liste des Global Admins)
UserAuthenticationMethod.Read.AllLire les méthodes d'authentification enregistrées (MFA, FIDO2, authenticator app)
SecurityEvents.Read.AllLire le Microsoft Secure Score et les recommandations de sécurité
DeviceManagementManagedDevices.Read.AllLire les appareils gérés par Intune (conformité, chiffrement, OS)
DeviceManagementConfiguration.Read.AllLire les politiques de configuration et de conformité Intune
MailboxSettings.ReadLire les paramètres des boîtes mail (fuseau horaire, langue, délégations)
Mail.ReadBasicLire les règles de boîte de réception (détection de règles suspectes de transfert)
DelegatedPermissionGrant.ReadWrite.AllLire les applications OAuth tierces ayant un accès délégué sur le tenant
